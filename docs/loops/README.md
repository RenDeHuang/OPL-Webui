# Loops

- owner: loop owner
- purpose: 定义 MVP 闭环名称和责任。
- state: active
- machine boundary: 非机器接口；loop 状态和事件以 contracts 为准。

## MVP Loops

- Intake: 从用户目标生成可执行项目输入。
- Project: 保存项目上下文、成员、状态。
- Task Runtime: 调度任务、记录步骤、暴露运行状态。
- Review: 支持审阅、评论、批准、退回。
- Artifact: 管理正式交付物、版本、下载。
- Tenant: 管理租户、成员、权限、配额和隔离。
